# Включение Sentry (продакшн‑мониторинг)

> Стандарт: RULES.md (сначала анализ, потом реализация). В этом проекте инициализация уже подготовлена.

## Шаги включения

1. Установить пакет

```powershell
npm i @sentry/react --save
```

2. Добавить DSN в окружение

Скопируйте `docs/ENV_EXAMPLE.md` в `.env` и пропишите:

```
VITE_SENTRY_DSN=ваш_dsn_из_sentry
```

3. Проверить код инициализации

- `src/main.tsx` — `initSentry()` вызывается
- `src/main.tsx` — приложение обёрнуто в `SentryErrorBoundary` (готово)
- `src/shared/monitoring/sentry.ts` — при наличии DSN начнёт отправку событий

4. Сборка и проверка

```powershell
npm run build
npm run preview
```

## Проверка работоспособности

- Явно бросить ошибку из любого компонента и убедиться, что событие видно в Sentry
- Проверить Performance/Replay (если включены)

## Примечание

Без DSN Sentry остаётся в STUB‑режиме (без отправки). Код уже подготовлен для безопасного включения.
