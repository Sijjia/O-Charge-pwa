## Фаза 0 — Security Baseline (обязательная перед релизом)

Цель: убрать ключевые риски безопасности и консистентности аутентификации.

- [ ] Перейти на cookie‑based сессии (HttpOnly, Secure, SameSite=Lax/Strict)
- [ ] Добавить CSRF‑защиту и refresh rotation, короткий TTL access токена
- [x] Унифицировать auth‑хранилище (убрать дублирование сторов/expiry)
- [x] Workbox: не кэшировать запросы с `Authorization` (NetworkOnly)
- [x] Workbox: ограничить кэширование `*.supabase.co` (только публичные GET)
- [x] SW/Build‑хуки: убраны лог‑вызовы и пустые блоки; pre‑commit/pre‑push проходят
- [ ] Включить strict Content Security Policy (script-src nonce/sha, connect-src)
- [ ] Добавить Security Headers: Permissions‑Policy, Referrer‑Policy, X‑CTO, COOP/COEP
- [x] Включить Sentry (frontend) — STUB инициализирован, включится после установки пакета
- [ ] Финальный аудит секретов/переменных окружения/логов

Критерии готовности:

- [ ] Нет долгов по хранению токенов в `localStorage`
- [x] SW не кэширует приватные ответы
- [ ] Sentry подключен, ошибки агрегируются и алертятся
- [ ] Проверка угроз (XSS/CSRF/Token theft) пройдена

Операционный статус:

- [x] Typecheck/ESLint/Vitest — зелёные на main
- [x] Push в `origin/main` прошёл (CI pre‑push: typecheck/lint/test)

### Cookie‑Auth rollout checklist (staging → prod)

- [x] Подготовлен документ для стенда: `docs/STAGING_COOKIE_AUTH.md`
- [ ] Backend: готово `/api/v1/auth/login|refresh|logout|csrf` (HttpOnly cookies, rotation)
- [ ] Staging: `VITE_AUTH_MODE=cookie`, `VITE_ENABLE_CSRF=true`, `VITE_CSRF_TRUSTED_ORIGINS=<api_origin>`
- [ ] Smoke: `/auth/debug` — GET csrf, POST refresh, POST logout (200/401 по сценарию)
- [ ] GET‑запросы с 401 выполняют одноразовый refresh‑retry (включён флагом)
- [ ] Клиент не отправляет `Authorization: Bearer` в cookie‑режиме (подтверждено сниффером)
- [ ] CSRF‑токен передаётся в мутирующих запросах на доверенные origin
- [ ] Prod: включены CSP и security headers по инструкциям деплоя
