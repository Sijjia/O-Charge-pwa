2 OCPP Симулятор

Виртуальная зарядная станция, которая подключается по OCPP 1.6J как настоящая. Создаёт реальные сессии, считает энергию и стоимость.

Веб-интерфейс: redp.charge.redpay.kg/admin/simulator
Авто-старт: При вызове POST /charging/start клиентом — симулятор автоматически подключает EV и начинает зарядку. Ручные шаги (plug-in, start-charging) нужны только для пошагового тестирования.

API

ШагМетодEndpointОписание
1. ЗапускPOST/api/v1/admin/simulator/startСоздаёт станцию, подключает по WebSocket
2. КабельPOST/api/v1/admin/simulator/plug-inИмитация подключения EV (автоматически при RemoteStart)
3. СтартPOST/api/v1/admin/simulator/start-chargingStartTransaction, счётчик энергии растёт (автоматически при RemoteStart)
4. СтопPOST/api/v1/admin/simulator/stop-chargingStopTransaction, расчёт стоимости
5. ВыклPOST/api/v1/admin/simulator/stopОтключает станцию
—GET/api/v1/admin/simulator/statusТекущее состояние
—GET/api/v1/admin/simulator/logЛог OCPP-сообщений
—GET/api/v1/charging/activeАктивная сессия текущего пользователя (новый)

Пример

# Запустить симулятор
curl -b cookies.txt -X POST \
  -H "Content-Type: application/json" \
  -H "X-CSRF-Token: $CSRF" \
  -d '{"station_id": "SIM-TEST", "connectors": 2}' \
  https://ocpp.charge.redpay.kg/api/v1/admin/simulator/start

# Подключить кабель
curl -b cookies.txt -X POST \
  -H "Content-Type: application/json" \
  -H "X-CSRF-Token: $CSRF" \
  -d '{"connector_id": 1}' \
  https://ocpp.charge.redpay.kg/api/v1/admin/simulator/plug-in

# Начать зарядку
curl -b cookies.txt -X POST \
  -H "Content-Type: application/json" \
  -H "X-CSRF-Token: $CSRF" \
  -d '{"connector_id": 1, "id_tag": "SIM-USER-001"}' \
  https://ocpp.charge.redpay.kg/api/v1/admin/simulator/start-charging

3 Webhooks (колбэки)

Система отправляет HTTP POST на ваш URL при каждом событии зарядки или изменении статуса.

Регистрация webhook

curl -b cookies.txt -X POST \
  -H "Content-Type: application/json" \
  -H "X-CSRF-Token: $CSRF" \
  -d '{
    "name": "My Production Webhook",
    "url": "https://your-server.com/webhook/redpetroleum",
    "secret": "your-hmac-secret-min-8-chars",
    "events": [
      "connector.status_changed",
      "charging.started",
      "charging.progress",
      "charging.completed",
      "charging.error",
      "station.online",
      "station.offline"
    ]
  }' \
  https://ocpp.charge.redpay.kg/api/v1/admin/webhooks

Формат POST на ваш URL

{
  "event": "charging.started",
  "timestamp": "2026-03-25T10:00:00+00:00",
  "data": {
    "station_id": "st-001",
    "connector_id": 1,
    "session_id": "uuid-session",
    "transaction_id": 12345,
    "meter_start": 1000
  }
}

Заголовки

ЗаголовокЗначение
Content-Typeapplication/json
X-Webhook-EventТип события (например charging.started)
X-Webhook-Signature-256sha256={HMAC-SHA256 hex digest}

Верификация подписи

# Python
import hmac, hashlib

signature = hmac.new(
    b"your-hmac-secret",
    request.body,
    hashlib.sha256
).hexdigest()

expected = request.headers["X-Webhook-Signature-256"].replace("sha256=", "")

if not hmac.compare_digest(signature, expected):
    return 403

Retry и throttling

  • Retry: 3 попытки с exponential backoff (1с → 5с → 25с)
  • Успех: HTTP 2xx = доставлено, иначе retry
  • Throttle: charging.progress отправляется не чаще 1 раза в 30 секунд

Все события

СобытиеКогдаКлючевые поля в data
connector.status_changedСтатус коннектора изменилсяstation_id, connector_id, status, error_code
charging.startedЗарядка началасьstation_id, connector_id, session_id, transaction_id, meter_start
charging.progressОбновление показанийstation_id, session_id, energy_kwh, power_kw
charging.completedЗарядка завершенаstation_id, session_id, energy_kwh, meter_stop, reason
charging.errorОшибка OCPPstation_id, connector_id, error_code, info
station.onlineСтанция в сетиstation_id, serial_number
station.offlineСтанция оффлайнstation_id, serial_number

Управление

GET/api/v1/admin/webhooks — список подписок
POST/api/v1/admin/webhooks — создать подписку
GET/api/v1/admin/webhooks/{id} — детали
PUT/api/v1/admin/webhooks/{id} — обновить
DEL/api/v1/admin/webhooks/{id} — удалить
POST/api/v1/admin/webhooks/{id}/test — тестовая отправка
GET/api/v1/admin/webhooks/{id}/logs — логи доставки
